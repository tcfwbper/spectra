## Overview

SessionMetadata represents the persistable state of a session, excluding the event history. It captures the core identifying information, lifecycle status, timestamps, current workflow position, session-scoped data store, and any fatal error. SessionMetadata is designed as a serializable structure that can be persisted independently of the full Session entity. It does not manage its own persistence; that responsibility belongs to SessionMetadataStore.

## Behavior

1. SessionMetadata is a plain data structure with no methods of its own.
2. All mutations to SessionMetadata fields occur through Session entity methods (see [Session](./session/session.md)).
3. SessionMetadata is embedded in the Session entity, allowing direct field access on Session instances.
4. SessionMetadata instances are created only during Session construction by SessionInitializer.
5. SessionMetadata is designed to be JSON-serializable for persistence to `session.json`.
6. The Error field uses the `omitempty` JSON tag, meaning it is omitted from serialization when nil.

## Inputs

SessionMetadata is not invoked; it is constructed as part of Session initialization. See [Session](./session/session.md) for construction details.

## Outputs

### SessionMetadata Struct

| Field | Type | Constraints | Description |
|---|---|---|---|
| `ID` | string (UUID) | Generated by SessionInitializer; unique across all sessions | Unique session identifier |
| `WorkflowName` | string | Non-empty, PascalCase | Name of the workflow being executed |
| `Status` | string | Enum: `"initializing"`, `"running"`, `"completed"`, `"failed"` | Current execution status |
| `CreatedAt` | int64 (POSIX seconds) | > 0 | Timestamp at construction |
| `UpdatedAt` | int64 (POSIX seconds) | >= `CreatedAt` | Timestamp of last in-memory mutation; refreshed by every mutating method |
| `CurrentState` | string | Always a workflow-defined node name | Active node in the workflow state machine |
| `SessionData` | `map[string]any` | See [Session data.md](./session/data.md) for namespace conventions and ClaudeSessionID type rules | Session-scoped key-value store |
| `Error` | `*AgentError` or `*RuntimeError` (nullable) | Nullable; non-nil iff `Status == "failed"` | First fatal error (first-error-wins) |

## Invariants

1. **Type Integrity**: SessionMetadata is a value type (struct) with no embedded locks or channels. It is safe to copy for serialization.

2. **Status Enumeration**: `Status` must always be one of the four defined values: `"initializing"`, `"running"`, `"completed"`, or `"failed"`.

3. **Timestamp Ordering**: `0 < CreatedAt <= UpdatedAt` at all times.

4. **Error Correlation**: `Status == "failed"` iff `Error != nil`.

5. **Non-Empty ID**: `ID` must always be a valid UUID v4 string.

6. **Non-Empty WorkflowName**: `WorkflowName` must always be a non-empty string in PascalCase.

7. **Non-Empty CurrentState**: `CurrentState` must always be a non-empty string representing a workflow-defined node name.

8. **SessionData Never Nil**: `SessionData` must never be nil; it must be initialized as an empty map `map[string]any{}` if there are no entries.

9. **JSON Serialization Compatibility**: All field types must be JSON-serializable. The Error field's dynamic type (`*AgentError` or `*RuntimeError`) must implement JSON marshaling.

10. **Error Field Omitempty**: When Error is nil, it must be omitted from JSON serialization (enforced via `omitempty` JSON struct tag).

## Edge Cases

- **Condition**: SessionMetadata is copied for serialization while Session methods are mutating it.
  **Expected**: This is prevented by Session's locking model. SessionMetadataStore must acquire the Session's read lock before copying SessionMetadata for serialization.

- **Condition**: `UpdatedAt` is older than `CreatedAt` due to clock skew or manual modification.
  **Expected**: This violates the timestamp ordering invariant. Session methods must ensure `UpdatedAt` is always set to `max(UpdatedAt, now())` on each mutation.

- **Condition**: `Status` is set to a value outside the enum (`"initializing"`, `"running"`, `"completed"`, `"failed"`).
  **Expected**: This violates the status enumeration invariant. Session methods must only set `Status` to one of the four defined values.

- **Condition**: `Error` is non-nil but `Status` is not `"failed"`.
  **Expected**: This violates the error correlation invariant. Session's `Fail` method must atomically set both `Error` and `Status == "failed"`.

- **Condition**: `SessionData` is nil.
  **Expected**: This violates the SessionData never-nil invariant. SessionInitializer must initialize it to an empty map. Session methods must never assign nil to SessionData.

- **Condition**: `Error` field holds a type other than `*AgentError` or `*RuntimeError`.
  **Expected**: This violates the type constraint. Session's `Fail` method must reject errors that do not match these types.

- **Condition**: JSON serialization encounters a value in `SessionData` that cannot be marshaled (e.g., a channel or function).
  **Expected**: SessionMetadataStore's serialization will fail with a JSON marshaling error. Session methods should validate SessionData entries to ensure they are JSON-serializable.

- **Condition**: `Error` field is nil and JSON serialization is requested.
  **Expected**: The `error` field is omitted from the JSON output entirely (no `"error": null` entry).
