# Event

## Overview

An Event is a typed signal that drives state transitions in the workflow state machine. Each event carries a workflow-defined type identifier, an optional message string, and an optional JSON payload. Events are emitted by agents or humans, and are stored in the session's event history in chronological order. Events are the only mechanism for workflow progression. The source of the event is captured in `EmittedBy`, which is automatically set by the Runtime to the session's current node name (`Session.CurrentState`) at the time of event emission.

## Behavior

1. An event is created when an agent or human invokes `spectra-agent event emit <type> --session-id <UUID> [--claude-session-id <UUID>] [--message <string>] [--payload <json>]`.
2. The `--session-id` flag is mandatory and used to find the runtime socket. If omitted, the spectra-agent CLI can't find the runtime socket.
3. If the user omits the `--message` flag, the runtime automatically sets `Message` to an empty string `""`.
4. If the user omits the `--payload` flag, the runtime automatically sets `Payload` to an empty JSON object `{}`.
5. The runtime assigns a unique UUID to the event, records the current POSIX timestamp, and automatically sets `EmittedBy` to the session's `CurrentState` (the current active node name).
6. The event type must match one of the types defined in the workflow's transition rules.
7. The runtime validates that the session is in `Status == "running"` before accepting the event. Events are rejected if the session is in `"initializing"`, `"completed"`, or `"failed"` status.
8. The runtime evaluates the event against the current state's transition rules to determine the next state.
9. The event is appended to the session's `EventHistory` in chronological order.
10. The event remains immutable once recorded.
11. The runtime delivers the `Message` field content to the recipient determined by the workflow's routing for that event type.

## Inputs

| Field | Type | Constraints | Required |
|-------|------|-------------|----------|
| Type | string | Non-empty, must follow Go naming conventions (PascalCase), must be defined in the workflow | Yes |
| Message | string | Any valid string | No (defaults to `""`) |
| Payload | JSON object | Valid JSON object, must not be a primitive or array | No (defaults to `{}`) |
| SessionID | string (UUID) | Must reference an existing session | Yes |

**Note**: `EmittedBy` is not provided by the caller. It is automatically set by the Runtime to the value of `Session.CurrentState` (the current active node name) at the time of event emission.

## Outputs

### Event Structure

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| ID | string (UUID) | Generated by runtime, unique across all events | Unique identifier for the event |
| Type | string | Non-empty, PascalCase, workflow-defined | Event type identifier |
| Message | string | Any valid string, defaults to `""` | Optional message for the event recipient |
| Payload | JSON object | Valid JSON object, defaults to `{}` | Event-specific data |
| EmittedBy | string | Workflow-defined node name, automatically set by Runtime | Node name from which the event was emitted (set to `Session.CurrentState` at emission time) |
| EmittedAt | int64 (POSIX timestamp) | Positive integer | Timestamp when the event was emitted |
| SessionID | string (UUID) | Valid UUID, references an existing session | Associated session identifier |

## Invariants

1. **Type Validation**: `Type` must be a workflow-defined event type. System-reserved event types do not exist in this design.

2. **Message String Compliance**: `Message` must always be a valid string. It may be empty (`""`), but must not be null.

3. **Payload JSON Compliance**: `Payload` must always be a valid JSON object `{}`. It must not be a JSON primitive (string, number, boolean, null) or array. It must not be null.

4. **EmittedBy Automatic Assignment**: `EmittedBy` is automatically set by the Runtime to the value of `Session.CurrentState` at the time of event emission. This field must not be provided by the caller.

5. **Session Reference Integrity**: `SessionID` must reference an existing session at the time of event emission.

6. **Session Status Requirement**: Events may only be emitted when the session's `Status == "running"`. Events emitted during `"initializing"`, `"completed"`, or `"failed"` status must be rejected.

7. **Event Immutability**: Once an event is created and appended to `EventHistory`, none of its fields may be modified.

8. **Chronological Ordering**: Events in `EventHistory` must be ordered by `EmittedAt` in ascending order.

## Edge Cases

- **Condition**: Event type is not defined in the workflow.
  **Expected**: The runtime must reject the event emission and return an error. The event must not be recorded.

- **Condition**: User omits `--message` flag when invoking `spectra-agent event emit <type> --session-id <UUID>`.
  **Expected**: The runtime must automatically set `Message` to an empty string `""` and accept the event.

- **Condition**: `Message` contains a non-string value (detected during parsing).
  **Expected**: The runtime must reject the event emission with a validation error indicating that `Message` must be a string.

- **Condition**: User omits `--payload` flag when invoking `spectra-agent event emit <type> --session-id <UUID>`.
  **Expected**: The runtime must automatically set `Payload` to an empty JSON object `{}` and accept the event.

- **Condition**: `Payload` is an invalid JSON string.
  **Expected**: The runtime must reject the event emission and return a JSON parse error. The event must not be recorded.

- **Condition**: `Payload` is a JSON primitive (e.g., `"string"`, `123`, `true`, `null`) or array (e.g., `[]`).
  **Expected**: The runtime must reject the event emission with a validation error indicating that payload must be a JSON object.

- **Condition**: `Payload` is `null`.
  **Expected**: The runtime must reject the event emission with a validation error indicating that `Payload` must not be null. It must be a JSON object.

- **Condition**: `SessionID` references a non-existent session.
  **Expected**: The runtime must reject the event emission with a "session not found" error.

- **Condition**: `SessionID` references a session with `Status == "initializing"`.
  **Expected**: The runtime must reject the event emission with a "session not ready" error.

- **Condition**: `SessionID` references a session with `Status == "completed"` or `Status == "failed"`.
  **Expected**: The runtime must reject the event emission with a "session terminated" error.

- **Condition**: Two events are emitted simultaneously (same `EmittedAt` timestamp).
  **Expected**: The runtime must serialize event processing to ensure deterministic ordering in `EventHistory`. If timestamps collide, use `ID` lexicographic order as a tiebreaker.

- **Condition**: A transition moves the session to a new node, and the `Message` field contains a notification for that node.
  **Expected**: The runtime queues the `Message` and delivers it when the target node becomes active (i.e., when `CurrentState` transitions to that node). If the session terminates before the node becomes active, the notification remains in the session log but is marked as undelivered.
